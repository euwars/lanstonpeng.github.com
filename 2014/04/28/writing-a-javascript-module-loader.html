var Util = Loader.import("Util");

我们的其中一个重点就是实现Loader.import方法

问题1:

是否能动态同步加载Util模块呢，即在加载Util之前，console.log是不会运行

答案是否定的，我们可以回顾一番javascript加载脚本的方式

  • 创建script标签
  • 通过ajax请求代码字符串，动态解析字符串
  • WebWorker(本质上与动态请求字符串差异不大)

上述几种方式都无法做到同步的方式， 那么，展开思路， 若我们坚持使用这样同步写法 我们可以寻找一种方式，在运行Loader.import('Util')的时候，实际上Util模块已经加载完毕，这个模块加载过程运行整个程序前已经完成 若我们需要动态加载模块，所有分析加载都在运行时,其中可以做到的形式或许会长成这么一个样子

    Loader.import("Util",function(Util){
        console.log(Util);
    });

上面的写法是否与requireJS有点类似呢? 为了更好的理解其工作原理，我们可以先从实现一个简单的异步加载器开始

var Loader = (function(window){
    var document = window.document;
    var config = {
        modulePath:"./"
    };

    function importModule(moduleName,callback){
        var script = document.createElement("script");
        script.src = config.modulePath + moduleName + ".js";
        document.body.appendChild(script);
        script.onload = function(){
            callback && callback.call(this);
        }
    }
    return {
        import:importModule
    }
})(window);

如果就这么运行，会crash，因为在callback中，我们没有在callback.call(this)中指定a这个对象，那么在a.js里面会调用undefined.say当然就会出错了

ok,那么问题就可以这么描述 如何在script.onload的里面的callback带上a这个对象呢

script.onload = function(){

既然要拿到a这个对象，那么，我们就不能直接像上面在全局定义a,至少需要返回给某个对象，以供后面使用，所以，我们可以按照这个思路看看,按照大家稍微熟悉的语法
